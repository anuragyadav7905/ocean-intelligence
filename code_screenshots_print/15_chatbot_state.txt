15_chatbot_state.txt — Chatbot.jsx: component state & auto-scroll
==================================================================

What this screen shows
----------------------
The opening of the `Chatbot` React component. It sets up refs,
component state, and a side-effect that keeps the message list
scrolled to the latest reply.

Line-by-line walkthrough
------------------------
Line 58       `export default function Chatbot()` — default export
              that the parent page renders.
Line 59       `useRef(null)` — `ref` is attached to the chatbot's
              root section so we can detect when it scrolls into
              view.
Line 60       `useInView(ref, ...)` returns a boolean that flips to
              true the first time the section enters the viewport.
              Options:
                • `once: true` — only fire the first time, never
                  reset.
                • `margin: '-80px'` — trigger 80 px before fully in
                  view, so the entrance feels eager rather than
                  late.
Line 61       `scrollRef` is attached to the scrollable message
              container; we'll use it inside `useEffect` to keep the
              chat pinned to the bottom.
Lines 63–65   Three pieces of component state:
                • `messages` — the chronological message list,
                  seeded from `INITIAL_MESSAGES` (a welcome script).
                • `input`    — the controlled value of the text box.
                • `loading`  — true while a request is in flight; the
                  UI uses it to disable the send button and show a
                  thinking indicator.
Lines 67–71   Auto-scroll side-effect. Whenever `messages` or
              `loading` changes, sets the container's
              `scrollTop = scrollHeight`, which jumps to the bottom
              of the conversation. Tying the effect to both deps
              means it fires both when a message arrives **and**
              when the typing indicator appears/disappears, keeping
              the latest content visible without the user having to
              scroll.

Why it matters
--------------
These ~13 lines establish everything the rest of the component
relies on: state for messages, scroll behaviour for UX polish, and
a viewport ref for the entrance animation. The next screenshot
(`sendMessage`) actually calls the API and pushes new messages
into this state.
