07_nlp_query.txt — nlp_engine.py: query() — similarity matching
================================================================

What this screen shows
----------------------
The public `query` method. Given a user's message it returns the
best-matching knowledge-base answer along with category and
confidence. This is what `app.py` calls per request.

Line-by-line walkthrough
------------------------
Line 64       Signature: takes the raw user query and returns a dict.
Line 66       Cleans the query through the same `_preprocess` used at
              fit time so the vector spaces line up.
Line 67       Transforms the cleaned query into a TF-IDF row vector
              using the *already-fitted* vectoriser.
Line 68       Computes cosine similarity between that single query
              vector and every row of the TF-IDF matrix, then
              flattens to a 1-D NumPy array of scores.
Line 69       Picks the index of the highest-scoring document.
              `int(...)` is just to convert from numpy.int64 so the
              JSON serialiser is happy.
Line 70       Pulls the matching score out as a plain float.

Lines 72–83   **Low-confidence safety net.** If even the best match
              scores below 0.05 the model genuinely doesn't know.
              Rather than confidently returning a wrong answer, the
              engine returns a friendly fallback message that nudges
              the user toward topics the bot does cover (marine
              animals, coral reefs, currents, fisheries, climate
              change, biodiversity) and mentions that the AEGFA
              pipeline keeps expanding the KB. Category becomes
              `unknown` and `matched_question` is None so the UI can
              display a different state.

Lines 85–91   **Happy path.** Looks up the metadata for the winning
              index and returns the answer, category, calibrated
              confidence, and the original matched question. The
              calibration `min(confidence * 1.2, 0.99)` slightly
              boosts cosine scores into a range users find more
              intuitive while clamping at 0.99 so the model never
              claims certainty.

Why it matters
--------------
This single method is the project's retrieval layer. It is fast
(one vector transform + one matrix multiply), deterministic, and has
a built-in "I don't know" branch so the chatbot degrades gracefully
when asked about something outside its corpus.
